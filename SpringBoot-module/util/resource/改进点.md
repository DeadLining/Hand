1. `@Permission`使用租户层(不要用平台层)

   ```java
   @Permission(level = ResourceLevel.ORGANIZATION)
   ```

2. `@RequestMapping`命名时使用`'-'`，不要使用`'_'`

   ```java
   @PutMapping("/update-header")
   ```

3. DDD层级之间的调用关系：

   ```java
   Controller <—— Service <—— Repository <—— Mapper
   ```

4. 乐观锁的处理：

   - 方法一

   先从数据库中读取出对应的数据记录（A类对象），然后对A进行setter()更新预处理，再进行updateSelective()更新操作（会自动地更新版本号）

   - 方法二（最好）

   从数据库中读取出来的不是类对象，而是对应的version版本号，将version设置到需要更新入数据库的类对象中，以防止脏数据的产生（乐观锁）

5. 当设置了Lov值集后，若想插入、修改数据记录时，需要将相应的值集字段设置成值集内存在的value值（即使你不使用它）

   ```java
   //error
   {
     "failed": true,
     "code": "[orderStatus {org.hzero.boot.platform.lov.annotation.LovValue.message}; ]",
     "message": "[orderStatus {org.hzero.boot.platform.lov.annotation.LovValue.message}; ]",
     "type": "warn"
   }
   ```

7. 在使用Api接口时，接收的参数最好使用`@RequestBody`注解，将需要读取的数据以json的格式读入
